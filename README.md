
### Тест 1 — Добавление задач (проверка ООП и типов)
1. Нажать `1` (Add task)
2. Добавить задачу типа **Simple**:
   - Task type: `1`
   - Title: `Buy milk`
   - Description: `2 liters`
   - Priority: `2`

3. Нажать `1` (Add task) снова
4. Добавить задачу типа **Deadline**:
   - Task type: `2`
   - Title: `Submit project`
   - Description: `University`
   - Priority: `3`
   - Due date: `2026-01-20`

5. Нажать `1` (Add task) снова
6. Добавить задачу типа **Recurring**:
   - Task type: `3`
   - Title: `Workout`
   - Description: `Gym`
   - Priority: `1`
   - Repeat every N days: `3`

✅ Ожидаемый результат: добавлены 3 разные задачи (разные типы).

---

### Тест 2 — Просмотр задач
1. Нажать `2` (List all tasks)

✅ Ожидаемо: выведется таблица со всеми задачами, где видно:
- ID
- Done (Yes/No)
- Priority (1–3)
- Type (Simple/Deadline/Recurring)
- Details (due date / interval)

---

### Тест 3 — “List open tasks” (только незавершённые)
1. Нажать `4` (Toggle done by ID)
2. Ввести ID любой задачи (например `1`)
3. Нажать `3` (List open tasks)

✅ Ожидаемо: задача с ID=1 исчезнет из списка open tasks (потому что стала Done).

---

### Тест 4 — Поиск по названию
1. Нажать `6` (Search by title)
2. Ввести: `work`

✅ Ожидаемо: найдётся задача `Workout` (поиск нечувствителен к регистру).

---

### Тест 5 — Сортировка
1. Нажать `7` (Sort by priority desc, id asc)
2. Нажать `2` (List all tasks)

✅ Ожидаемо: сначала выводятся задачи с Priority=3, затем 2, затем 1.

---

### Тест 6 — Сохранение и загрузка
1. Нажать `9` (Save)
   - Проект создаст файл `tasks.db` в папке запуска (рядом с exe или в рабочей директории).
2. Закрыть программу через `0` (Exit)
3. Запустить снова
4. Нажать `2` (List all tasks)

✅ Ожидаемо: задачи сохраняются между запусками (автозагрузка при старте из `tasks.db`).

---

## 6) Где хранится база задач
Файл: `tasks.db`  
Создаётся автоматически при сохранении или выходе.

Если удалить `tasks.db` — программа стартует с пустым списком задач.

---

## 7) Что демонстрирует ООП
- Абстрактный класс: `Task`
- Наследники:
  - `SimpleTask`
  - `DeadlineTask`
  - `RecurringTask`
- Полиморфизм:
  - виртуальные методы `typeName()`, `detailsLine()`, `serialize()`
- Безопасная работа с памятью:
  - `std::unique_ptr<Task>`
- Инкапсуляция:
  - поля задач закрыты, доступ через методы
